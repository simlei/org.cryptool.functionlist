static CfgTransfer(element: Any) → Any¶

Transfer the configuration (config.step) of a step to the parameters for invoking the step (params.step)

Naively identifies paths in the config to be augmented to the workspace by looking at whether a string starts with “./”

get_config_proc()¶

returns the context processor contributions for the step. describes how a dictionary provided by e.g. YAML config (string based) is transferred to the “params” space for the step.

For flist steps, this is implemented by looking at the step param prototype, and specifying for each subelement to be transformed according to FlistStep.CfgTransfer, where first-level lists are recursed into.
